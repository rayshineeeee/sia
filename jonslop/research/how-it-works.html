SIA Hackathon · Taste-Shader · What We Built

Self-improving an agent toward a person's taste — and what that honestly reveals.

A frozen model writes a GLSL shader blind; it doesn't match the target taste, and we can't RLHF its taste. So we use SIA: a Meta Agent writes a Task-Specific Agent, a Task Environment runs it and scores it on held-out taste, and an Improvement Agent rewrites the agent for the next generation. We run SIA-H — harness only; the model stays frozen. The honest result: SIA's loop works and is legible, but on a subjective target a smarter agent didn't reliably raise held-out taste — reward faithfulness, not agent cleverness, is the ceiling.

Prototype · the visual

converge.py

A standalone, watchable convergence loop (gap → edit → render → score → keep-best) toward one reference. Frozen model, fixed script. The demo visualization — not SIA.

Experiment · the study

faithfulness_probe.py

Per step: the CLIP proxy SIA optimises vs an independent VLM judge it never sees. Where they diverge, the loop is gaming its reward. Turns "auto-score ≠ taste" into an automatic detector.

1 · SIA — the actual framework loop (canonical architecture, mapped to taste-shader)

SIA's real architecture from the docs, applied to our task. SIA-H: weights frozen — the Improvement Agent modifies the agent's code (target_agent.py), not the model. Every generation is scored on held-out taste in data/private/.

What SIA contributes that converge.py alone cannot: held-out, ungameable scoring (a real gain, not memorisation — the answer to our Track-3 reward-faithfulness problem); it improves the agent, so gains generalise; and it is general across tasks and can also update weights (SIA-W+H) — which we did not run.

What SIA actually changed (verified in runs/)

Weights — untouched. Default --focus harness; no train.py, no LoRA adapter, no Tinker/Modal anywhere in the runs. SIA-H, not SIA-W+H — we report that plainly. Harness — rewritten every generation. The Improvement Agent reads the held-out score + agent_execution.json and rewrites the whole target_agent.py (run_2):

Gentarget_agent.pyHeld-out tasteWhat the Improvement Agent changed (from improvement.md)

This is genuine SIA-H, working. The harness changes are substantial and well-reasoned — the Improvement Agent's logic is real, not cosmetic. We are not doing weight updates, and we don't claim to.

2 · The convergence engine — what the Task-Specific Agent does each generation

The agent's inner behaviour (prototyped standalone in tools/converge.py): close the gap to one reference, edit by edit. SIA wraps many of these and rewrites the agent that runs it.

The honest distinction (our Track-3 point). The convergence engine scores against the same reference it edits toward — a proxy it can game. SIA's evaluate.py (Diagram 1) scores on held-out taste the agent never sees. The gap between the proxy climbing and held-out taste actually improving is the reward-faithfulness question we study.

What we found — honestly

The reportable result is not a big number. It's a finding about how self-improving loops behave on subjective targets.

ObservationWhat it means
Harness grew + got smarter, held-out taste did not follow: 58.8 → 58.9 → 45.0SIA's Improvement Agent made plausible improvements the held-out reward did not confirm — gen 3 was worse. A more sophisticated agent ≠ more taste.
The Improvement Agent flagged its own proxyVerbatim from improvement.md: "Selection is purely by pixel variance — high variance does NOT mean taste alignment." The loop noticed its internal proxy wasn't taste.
The ceiling is the reward, not the agentOn a fuzzy goal, self-improvement is bottlenecked by whether the metric faithfully measures the goal. The held-out eval is the only thing that catches a "larped" improvement.
So we measure faithfulness directly. faithfulness_probe.py scores every step with the CLIP proxy and an independent VLM judge that is never part of the reward — a Two-Eyes test (SOTARE #4: "the delta between the eyes IS the alignment gap"). Grounded in the research: "we improve on the things we can measure" (Eric Jang) — outer-loop verifiability is the limit of self-improvement; SOTARE #27 calls an ungrounded "win" larping, valid only if a separate eval-bench confirms it. Our probe is that bench.
