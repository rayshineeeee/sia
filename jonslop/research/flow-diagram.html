Taste-SIA — what we're building, and how each piece is verified

A swipe onboarding (16 diverse images: like/dislike) + your reference set become a CLIP taste benchmark. SIA rewrites a prompt-writer so a frozen LLM writes GLSL in your taste; ModernGL renders it free/local; the taste-proxy scores it. Below the flow: every feature → a verification gate you approve, plus how to run, watch, and debug. No fallbacks — one clean path.

Build cycles → verification gates

Each cycle ships one feature, proves it with a verification you sign off, then we move on. The DEBUG column is also the scope check — if the output isn't what the gate describes, it caught something out of scope.

How to run · watch · connect

  1. Onboard (taste): open jonslop/tasks/taste-shader/frontend/swipe.html → swipe 16 → writes likes/dislikes.
  2. Fit benchmark: .venv/bin/python tools/fit_taste.py → data/private/taste_proxy.npz.
  3. Run SIA: .venv/bin/sia run --task_dir …/taste-shader --max_gen 5 --sandbox none.
  4. Watch live: dashboard auto-starts at http://127.0.0.1:8000 (per-generation renders + scores).
  5. See result: before/after renders in runs/run_*/gen_*/ + curve.png from plot_curve.py.
No fallbacks (by request). One path only — gpt-oss on Nebius, ModernGL, CLIP. No chart parachute, no model fallbacks. We build cycle by cycle; each gate must pass before the next.
